# reverse-engineering-lab5 — README

## Мета

Знайти пароль для переданого **скомпільованого** виконуваного файлу. Пароль формується динамічно під час виконання програми за алгоритмом, захованим у бінарі. Ваше завдання — віднайти цей алгоритм, відтворити його в коді (наприклад, у Python) і отримати очікуваний пароль, при поданні якого бінар повертає **Access granted**.

---

## Що Ви отримуєте

У роздачі **лише виконувані файли** (бінарні):

* `variant01` … `variant10` — готові скомпільовані файли.

---

## Інструменти (обов'язкові / рекомендовані)

**Обов’язково:**

* Ghidra (декомпілятор) — для статичного аналізу.
* Python 3 — для відтворення алгоритму у скрипті та виведення очікуваного серіалу.

**Рекомендовано:**

* `strings`, `file`, `ldd`, `objdump`, `readelf` — швидкі перевірки бінарів.
* radare2 або rizin — CLI-аналіз.
* GDB/LLDB/x64dbg — для динамічного відлагодження (за потреби).

---

## Загальний алгоритм роботи

### 1) Підготовка

1. Розпакуйте отримані файли в робочу директорію.
2. Перевірте права та формат бінаря: `chmod +x variant01` (за потреби), `file variant01`.
3. Зробіть швидкі сканування: `strings variant01`, `ldd variant01` (щоб побачити залежності), `readelf -h/--sections` або `objdump -x` — ці кроки дають уявлення про тип бінаря (PIE/NON-PIE), архітектуру і видимі константи.

### 2) Імпорт у Ghidra

* Відкрийте бінар у Ghidra і запустіть авто-аналіз (залиште стандартні опції або адаптуйте під архітектуру).
* Ознайомтеся зі списками рядків (Strings), функцій і графів викликів.

### 3) Пошук точки валідації

* Знайдіть рядки на кшталт `Access granted` / `Access denied` → знайдіть їхні XREF → перейдіть у функцію, що відповідає за перевірку.
* З’ясуйте, де порівнюється введений пароль і що програма очікує як `expected`.

### 4) Декомпіляція і аналіз

* Відкрийте цільову функцію в Decompiler view.
* Виявляйте блок(и), що будують ключ/масив — досліджуйте операції (арифметичні, побітові, повороти, множення, XOR тощо).

### 5) Відтворення алгоритму

* Реалізуйте обчислення в окремому скрипті (наприклад, Python). Скрипт повинен:

  * Приймати необхідні константи або масиви (взяті з бінаря).
  * Виконувати ту саму послідовність операцій.
  * Виводити plaintext / очікуваний пароль.
* Переконайтеся, що при поданні отриманого серіалу бінару результат — **Access granted**.

### 6) Перевірка та звіт

* Запустіть бінар і подайте знайдений пароль — переконайтеся, що він працює.

### 7) (Опційно) Patch/loader

* Якщо хочете — можете додати runtime-патч або демонстраційний лоадер. Це опційно і має бути чітко вказано у звіті. Нагадуємо: основна вимога — відновлення алгоритму, а не патчинг.
